When updating or refactoring code, never leave old stuff in there for 'reference', delete that! we only want the new stuff. If really necessary we'll have the old stuff avaliable through git.

Generally keep answers short, the user prefers a back and forth instead which is hindered by long answers.

When writing python, always put import statements at the top of the file!

When running gcloud or az commands NEVER do updates and ALWAYS ONLY read commands. Every change on a cloud env needs to be done by the user.
